## Vault Conventions

### File Naming
- Use **Title-Case** with hyphens: `Agent-Loop.md`, `Docker-Setup.md`
- Section directories use **numbered prefixes**: `01-Architecture/`, `02-Agent-Profiles/`
- Docgen files use **kebab-case**: `extension-points-python.md`

### Frontmatter
Every content page must include YAML frontmatter:

```yaml
---
tags: [agentzero, <section-topic>]
aliases: [<short-name>]
---
```

### Navigation
Every page includes breadcrumb navigation at the top:

```markdown
[← Home](../00-Home.md) · [↑ README](../README.md) · [→ Next Section](../XX-Section/)
```

### Linking Strategy
- Use **relative Markdown links**: `[text](../path/to/file.md)`
- Do NOT use Obsidian `[[wikilinks]]` — they break on Gitea/GitHub
- All links must work in standard Markdown renderers

### Content Structure
Each page follows this pattern:

1. Frontmatter (tags, aliases)
2. Breadcrumb navigation
3. Title (H1)
4. Overview paragraph
5. Detailed sections (H2/H3)
6. Tables for reference data
7. Code examples in fenced blocks
8. Notes/warnings as callout-style paragraphs

## Branch Strategy

| Branch | Purpose |
|--------|----------|
| `main` | Published content |
| `development` | Integration branch for PRs |
| `feature/<name>` | Individual feature branches |

## Contribution Workflow

1. Fork the repository
2. Create a feature branch from `development`
3. Write/edit content following conventions above
4. Run quality checklist (see below)
5. Submit PR to `development` branch

## Quality Checklist

Before submitting a PR, verify:

- [ ] YAML frontmatter with tags present
- [ ] Breadcrumb navigation at top of page
- [ ] All relative links work (test in browser)
- [ ] No `----` placeholder content remaining
- [ ] Title-Case file name
- [ ] Tables and code blocks properly formatted
- [ ] No broken images or missing references
- [ ] Content is accurate for current Agent Zero version
